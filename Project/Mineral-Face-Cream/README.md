## 🚫 為何先不做 SPF 版本

SPF 功能先保留為未來擴充，原因：

1. **美國法規門檻高**：含 SPF 聲稱的產品在美國被 FDA 歸類為 **OTC Drug**（非化妝品），需符合 Sunscreen Monograph、進行 SPF in vivo 測試、Broad Spectrum 測試，並在符合 OTC drug cGMP（21 CFR Part 211）的工廠生產
2. **開發成本大幅增加**：OTC CMO 費用、法規測試費用遠高於一般化妝品
3. **時間拉長**：穩定性測試 + 法規測試通常需要 12–18 個月
4. **先驗證市場**：先以純礦物保濕面霜打入市場、驗證品牌與配方，成功後再加入 SPF 版本

## 🗺️ 開發路徑（從零開始）

### 第一階段：產品定義
- [ ] 確定目標客群與差異化定位
- [ ] 研究競品（美國礦物保濕面霜市場）
- [ ] 訂定成本目標與售價區間
- [ ] 確定核心訴求（礦物成分 / 敏感肌 / clean beauty 等）

### 第二階段：配方開發
- [ ] 找 cosmetic formulator 或 contract lab
- [ ] 配方初稿（乳化系統、礦物成分選定、膚感調整）
- [ ] 小批次打樣 + 使用者測試

### 第三階段：安全與穩定性測試
- [ ] 穩定性測試（加速老化 / 室溫 12 個月）
- [ ] 皮膚刺激性 / 安全評估（HRIPT 或 patch test）
- [ ] 微生物挑戰測試（preservative efficacy）

### 第四階段：製造與品管
- [ ] 找符合 cosmetic GMP（ISO 22716）的 CMO
- [ ] 建立 batch record 系統
- [ ] 包材選定與相容性測試

### 第五階段：上市準備
- [ ] 美國化妝品標示（MOCRA 2022 合規）
- [ ] 品牌 / 包裝設計
- [ ] 電商通路規劃（DTC / Amazon）
- [ ] 建立消費者回饋機制
